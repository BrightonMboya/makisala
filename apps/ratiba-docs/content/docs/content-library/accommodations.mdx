Open **Content Library** then **Accommodations** to manage this list.

## Fields

Each accommodation has:

- **Name.** The trading name as the lodge calls itself.
- **Location.** Park or region. Used as a filter in the itinerary builder and to drive default park fees.
- **Country.** For multi-country operators.
- **Description.** One to three paragraphs. Shown verbatim on proposals.
- **Star rating** (1 to 5) or a custom style tag (luxury, deluxe, mid, budget).
- **Room types.** A list of room categories the lodge offers (Twin, Double, Triple, Family). Used by the rate card.
- **Meal plans available.** Which of RO, BB, HB, FB the lodge supports. Used by the rate card.
- **Images.** Drag-and-drop upload, or paste a URL.
- **Tags.** Free-form tags for filtering.
- **Status.** Active or archived.

## Room types

Add only the room categories you actually book. If a lodge has 14 rooms but you only ever sell "Tent" and "Family Tent", just add those two.

The names you use here flow through to:

- The rate-card row matcher (the rate must exist for the named room type).
- The proposal's "accommodation" section.

Match the names to your supplier's rate sheet to avoid manual mapping.

## Linking to the rate card

After you create the accommodation, click **Add rate** to jump to the corresponding row in [Accommodation rates](/docs/rate-cards/accommodation-rates). Rates and accommodations are linked by ID, so renaming the accommodation later does not break existing rates.

## Bulk operations

- **Duplicate.** Right-click an accommodation to clone it (useful for related camps in the same group).
- **Bulk archive.** Filter the list, select multiple, and archive in one click.
- **Bulk re-tag.** Same flow — filter, select, apply a tag.

## What clients see

On proposals, accommodations show:

- Name and location.
- Lead image plus a gallery.
- Description.
- The room type and meal plan they're booked into for that day.

If you want the proposal to show a custom write-up that differs from your library copy, you can override the description per-itinerary in the day editor.
