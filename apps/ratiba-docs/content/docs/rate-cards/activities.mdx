title: Activity rates
description: Per-person or per-group prices for paid activities.
---

Activity rates correspond to the activities defined in your [Content Library](/docs/content-library/activities). The library holds the description, this tab holds the price.

## Fields

- **Activity.** Picks from your library.
- **Season.** Optional; if blank, applies year-round.
- **Pricing basis.** Per person or per group (must match the basis set on the activity).
- **Rate.**
- **Currency.**
- **Minimum pax.** Optional. If the activity requires a minimum (e.g. "Balloon flight requires 4 pax"), set it here. Ratiba will warn if you add it to a trip with fewer pax.
- **Maximum pax.** Optional capacity limit.

## Per-person vs per-group

Most safari add-ons are per-person: balloon flights, walking safaris, bush dinners. A few are per-group: a private chef night, a vehicle hire for a cultural visit.

The basis is locked on the activity itself (in the library), so the rate card row inherits it. If you change basis on the library entry, the rate card row updates accordingly.

## Lodge-included activities

If a lodge includes an activity in its rate (e.g. all game drives), don't add an activity-rate row for it. Either:

- Don't add it as an explicit activity (most common). The day implicitly includes the lodge's activities.
- Add it to the library marked as **Included**, and place it on the day for visibility. It will appear on the proposal but cost $0.

## Group vs FIT pricing

For activities with different rates above and below a pax threshold (e.g. "Cultural tour: $50pp for 1-3 pax, $35pp for 4+"), add two rate rows with min/max pax. Ratiba picks the row matching the trip's pax count.

## Updating after season change

Activity rates often go up slightly each year. Use the bulk-update tool: filter by season, then multiply all rates by 1.05 (or whatever the inflation is).
